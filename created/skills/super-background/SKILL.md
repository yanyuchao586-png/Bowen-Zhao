# 超级背调专家 v4.5.0（小蜂学掌定制版 · 聊天 + 文档版）

> **架构原则**：真实性优先 · 禁止编造 · 禁止推测 · 找不到写「暂未找到」· 先背调，后成文档 · 模板内置 · 脚本驱动 · 首次确认偏好 · 文档模式才补齐 Python · 不做 Excel

---

## 🚨 最高铁律：真实性

| 规则 | 说明 |
|---|---|
| **禁止编造** | 任何字段若无法通过 `web_search` / `web_fetch` 实际查到，必须填 **`暂未找到`**，绝对禁止猜测或推断填写 |
| **禁止推测邮箱** | 不得根据邮箱格式推算高管邮箱，除非明确标注 `(推测，未验证)` |
| **禁止推测 LinkedIn** | 不得凭名字+公司名拼 LinkedIn slug |
| **禁止推测 WhatsApp** | WhatsApp 仅从官网 Contact 页 / LinkedIn Contact Info / 用户提供的名片三个来源填写，其余一律填 `暂未找到` |
| **链接必须验证** | 所有 URL 必须经过 `web_fetch` 实测确认可访问且内容匹配，否则不得写入 |
| **社媒链接必须含账号路径** | 社媒链接必须精确到账号页（如 `instagram.com/fonrocheus`），**严禁填写平台首页**（如 `instagram.com` / `facebook.com` / `linkedin.com`）。填写平台首页 = 未找到，必须改写为 `暂未找到` |
| **没有就没有** | 找不到的内容不得用任何方式绕过：不填「该公司未公开此信息」等模糊表述，统一填 `暂未找到` |

## 🎯 核心定位

**输入**：公司名 OR 官网 URL OR 截图线索  
**输出**：两种模式任选其一
1. **聊天模式**：聊天框直出 10 维全息档案 + 决策人图谱
2. **文档模式**：按模板生成 Word 报告（如需 PDF，可在 Word 基础上再另存/转换）

## 🚦 执行流程

### Step 0 — 首次偏好确认

第一次使用本 skill 时，如果还不知道用户偏好，先用两问快速确认；若 `runtime/user-preferences.json` 已存在且版本匹配，则直接沿用，不再追问。

1. 回复模式：
   - 快速回复：优先给客户可用的结论，速度最快，适合正在沟通中的客户
   - 深度回复：维度更全、信息更细，适合高价值客户或二次复盘
2. 输出形式：
   - 纯文字：最快，不依赖本地环境
   - Word 文档：按内置模板出报告，便于转发和归档

默认偏好：
- 未明确回答时，默认“快速回复 + 纯文字”
- 如果用户明确说“按模板做成文档 / Word / 报告文件”，自动切到 Word 模式
- 如果用户明确说“先快点回复”，自动切到快速回复

记录规则：
- 将本次选择写入 `runtime/user-preferences.json`
- 后续同一用户/同一安装环境优先复用，不反复询问
- 只有当用户明确改口，才覆盖旧偏好

### Step 5 — 按模板生成文档

如果用户明确要求「按模板做成文档」「做成 Word」「生成报告文件」，在完成背调后，继续执行文档化流程：

1. 采用固定报告模板输出
   - 封面
   - 背调摘要
   - 10 维全息档案
   - 决策人图谱
   - 深度补充情报
   - 结论与行动建议
   - 资料来源
2. 文件格式优先级
   - 默认：`.docx`
   - 如需 PDF：先生成 Word，再按用户侧工具链另存/转换
3. 文件命名
   - 中文友好命名：`{公司名}_客户超级背调报告.docx`
   - 如遇编码问题，先用 ASCII 临时文件名生成，再重命名
4. 版式要求
   - 标题清晰
   - 表格整齐
   - 中文字体可读
   - 明确标注「暂未找到」
5. 交付前验证
   - 重新读取文件，确认中文不乱码
   - 确认文件能被正常打开
6. 模板来源
   - 固定使用 `assets/customer-background-report-template.docx`
   - 如需更新模板，优先替换该资产，不要要求用户重复提供模板文件
7. 生成脚本
   - 仅在 Word 模式触发时运行 `scripts/bootstrap_python.ps1`
   - 该脚本会检查 `runtime/bootstrap.json`，已完成则直接跳过
   - 若缺少 Python，则自动补齐；随后安装 `python-docx`
   - 使用 `scripts/build_report.py`
   - 输入建议为结构化 JSON（背调摘要、10 维内容、决策人图谱、来源、免责声明）
   - 默认输出到 `outputs/{公司名}_客户超级背调报告.docx`

#### 文档模式推荐提示词

```text
帮我继续按 super background 技能完成这个客户的背调，并把结果按模板做成 Word 文档。
要求：
1. 先做真实性核验，找不到的信息统一写“暂未找到”。
2. 背调完成后，生成和模板一致的报告结构。
3. 输出到 outputs 目录。
4. 如果中文文件名有编码问题，先用英文临时名生成，再重命名。
5. 生成后回读校验，确保中文不乱码。
```

### Step 6 — 模板化输出约定

固定模板的版式和章节顺序以资产文档为准。若用户要求“按你给的那个 Word 模板做”，默认含义如下：

1. 复用模板章节顺序和表格样式。
2. 将本次背调内容填入同结构章节。
3. 以 `{公司名}_客户超级背调报告.docx` 命名输出。
4. 若中文文件名受环境限制，先用英文临时名生成，再重命名。
5. 如需 PDF，先出 Word，再按用户侧工具链另存/转换。

### Step 7 — 模板资产读取

生成 Word 文档时，先读取 `assets/customer-background-report-template.docx` 作为版式基准。优先复用它的标题层级、章节顺序、表格风格和免责声明结构；只替换本次客户内容，不要让用户重复上传模板。

### Step 8 — 脚本化导出

当需要稳定导出 Word 时，先把本次背调整理成结构化 JSON，再调用 `scripts/build_report.py`。脚本负责把 JSON 渲染成 Word，模板资产负责提供页面尺寸、标题层级与表格风格。

## 📐 字段处理规则

| 情况 | 处理方式 |
|---|---|
| 该字段有公开、可验证的信息 | 直接填写，标注来源 |
| 该字段无公开信息 / 无法查到 | 填 **`暂未找到`** |
| 邮箱仅为格式推测 | 填 `xxx@domain.com (推测，未验证)` |
| 社媒链接只找到平台首页 | 填 `暂未找到`，禁止填 `instagram.com` 等平台域名 |
| LinkedIn URL 未实测 | 不填，写 `暂未找到` |
| WhatsApp 无官方来源 | 填 `暂未找到`，禁止猜号码 |
| 营收、进口额等数字无可信来源 | 填 `暂未找到`，**禁止估算** |

## ⛔ 绝对禁止

- ❌ 凭经验「估算」任何数字
- ❌ 凭姓名+公司名拼 LinkedIn URL
- ❌ 社媒字段填写平台首页（`instagram.com` / `facebook.com` / `tiktok.com` 等），必须精确到账号路径或填 `暂未找到`
- ❌ 凭邮箱格式推算高管邮箱（不加标注时）
- ❌ 凭公司名猜 WhatsApp / 手机号
- ❌ 生成任何 Excel / ZIP 文件；Word 仅可在文档模式下通过模板资产 + `scripts/build_report.py` 生成
- ❌ 调用任何外部 Python 脚本；仅允许使用技能自带的 `scripts/build_report.py` 和校验脚本
- ❌ 输出「该信息属于商业机密」「该公司未公开此数据」等绕过表述，一律改写为 `暂未找到`

- **v4.0.1（2026-06-03）**：**社媒链接账号路径强制校验**。新增铁律：社媒字段必须精确到账号页 URL（如 `instagram.com/fonrocheus`），填写平台首页（如 `instagram.com`）视同「暂未找到」，强制改写；同步更新「绝对禁止」和「字段处理规则」两处。
- **v4.0.0（2026-06-03）**：纯聊天框版。，保留 10 维全息档案 + 决策人图谱核心研究框架，直接在聊天框输出。新增严格的真实性字段规则（找不到 = 「暂未找到」，禁止推测）；决策人图谱改为「找到几人写几人」，不强制凑至 5 人；移除所有 sanity_check / excel_gen / validator 依赖。

- **v4.3.0（2026-07-02）**：新增 `scripts/build_report.py` 与结构化 JSON 导出流程。文档化输出改为模板资产 + 生成脚本组合，减少手工排版与乱码风险。
